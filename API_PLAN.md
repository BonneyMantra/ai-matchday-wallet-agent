# API Plan

## Stack

- Contracts: Solidity 0.8.24 with Foundry.
- App: Vite React TypeScript.
- Wallet / RPC: `viem` for local and X Layer JSON-RPC reads/writes.
- Data: local JSON proof artifacts generated by scripts; no hidden remote database.

## Contract API

`MatchdayAgentWallet` exposes:

- `submitPrediction(bytes32 matchId, uint8 pick, string rationale)` records the caller's pick and emits `PredictionSubmitted`.
- `resolveMatch(bytes32 matchId, uint8 result, string sourceLabel)` owner-only resolver path for hackathon/demo fixtures.
- `agentStats(address agent)` returns submitted/correct/settled counters.
- `predictionOf(address agent, bytes32 matchId)` returns pick details.

Picks are non-cash outcome choices:

- `0`: home
- `1`: draw
- `2`: away

## Script / CLI API

- `npm run rpc:smoke`: confirms X Layer RPC chain ID and block height.
- `npm run agent:pick`: creates a labeled agent decision artifact. Uses `OPENAI_API_KEY` only if present; otherwise writes `decisionMode: fixture-heuristic`.
- `npm run anvil:proof`: deploys locally, signs an agent prediction, resolves a fixture, and writes `proofs/local-anvil-proof.json`.
- `npm run deploy:xlayer`: deploys to X Layer testnet only when `DEPLOYER_PRIVATE_KEY` is present.
- `npm run agent:submit:xlayer`: submits a real agent prediction only when contract address and funded key are present.

## Fixture Policy

Seeded fixtures are allowed only as fixture/demo data. They must be labeled in:

- UI
- README
- `TRUTH_AUDIT.md`
- `QUALITY_GATE.md`
- proof JSON

No live sports-data API is required for the MVP because the hackathon idea is judged on X Layer state transition and wallet signing, not live oracle coverage. A future live-sports adapter can replace the fixture source without changing the contract state transition.
